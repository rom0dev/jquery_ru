Перед каждым проектом, будь то даже пет-проджект, необходимо собрать необходимые данные.

Цель: Бот должен оповещать об изменениях в расписании подписавшихся на класс.

Отсюда мы получаем:
* подписавшиеся - пользователи бота;
* класс - класс для которого есть изменение в расписании;
* расписание - данные содержащие класс, день, изменения.

Значит функционал бота будет таким:
* Получение изменении в расписании по таймеру;
* Регистрация пользователей (или удаление отметки "пользователь удалён");
* Добавление класса;
* Удаление класса;
* Отправка сообщениий об изменении в расписании с учётом пользователей, которые отписались от бота;
* Отписка от бота - soft delete;

Дополнительно хочу добавить:
* Отправка сообщения создателям бота;
* Отправка сообщений пользователям (по фильтру).
